# PBL Question Lab

**הופכים נושא לימודי לשאלת PBL שאי אפשר לפתור עם תשובה מגוגל.**

כלי AI למורים ומתכנני תוכנית לימודים — יוצר, מאבחן ומפתח שאלות מנחות לפרויקטי חקר, ומייצר תיקי פרויקט מוכנים לשימוש.

---

## מהו הכלי?

PBL Question Lab הוא כלי ווב המשלב בינה מלאכותית (Claude של Anthropic) עם מומחיות פדגוגית בלמידה מבוססת פרויקטים. הכלי מסייע למורים לנסח שאלות מנחות איכותיות — שאלות שמניעות חקירה אמיתית, יוצרות מתח ודילמה, ואינן ניתנות לפתרון על ידי חיפוש מהיר בגוגל או שאלה ל-ChatGPT.

---

## הבעיה הפדגוגית

שאלה מנחה (Driving Question) היא ליבת כל פרויקט PBL. שאלה טובה מניעה את התלמידים לחקור, לשאול שאלות חדשות ולייצר ידע — לא רק לאסוף מידע. אבל נסח שאלה כזו קשה בפועל:

- שאלות סגורות ("מה זה פוטוסינתזה?") מניבות עבודות ויקיפדיה.
- שאלות רחבות מדי ("איך נציל את כדור הארץ?") חסרות עוגן תוכני.
- שאלות ללא מתח אמיתי אינן מניעות חקירה עצמאית.

רוב המורים לא מקבלים הכשרה מספקת בניסוח שאלות מנחות, והתהליך יכול לקחת שעות של ניסוי וטעייה.

---

## מה הכלי עושה?

### 1. יצירת שאלות PBL

המורה ממלא פרטים בסיסיים — נושא, שכבת גיל, תחומי דעת, רמת נועזות — וה-AI מייצר שאלה מנחה מלאה:
- השאלה עצמה
- הסבר מדוע היא עובדת פדגוגית
- חוזקות ונקודות לשיפור
- שאלות משנה לחקירה
- רעיונות לתוצרים (לקהל מחוץ לכיתה)
- 2 ניסוחים חלופיים
- מבחן לחץ פדגוגי ב-10 קריטריונים

### 2. אבחון שאלה קיימת

המורה מדביק שאלה שכתב, והכלי מספק:
- ציון כולל (0–10)
- מה עובד ומה לא עובד
- הסיבה המרכזית לבעיה
- מה יקרה בפועל בכיתה אם עובדים עם השאלה הזו
- כיוון לשיפור + 2 ניסוחים חלופיים
- מבחן לחץ מלא

### 3. תיק פרויקט

לאחר בחירת שאלה (מיצירה או מאבחון), הכלי מייצר תיק פרויקט מוכן ליום הראשון:
- שם הפרויקט
- תיאור כולל למורה
- מטרות למידה ניתנות למדידה
- תכנים ומושגי מפתח
- שאלות משנה לחקירה
- 4 שלבי חקירה עם ציר זמן
- רובריקת הערכה (3 קריטריונים × 3 רמות)
- חוויית פתיחה

## מבחן הלחץ — 10 קריטריונים פדגוגיים

כל שאלה נבדקת אוטומטית לפי 10 קריטריונים (ציון 1–10 לכל קריטריון):

| קריטריון | מה הוא בודק |
|----------|-------------|
| **פתוחה** (`open_ended`) | האם השאלה מאפשרת מגוון תשובות לגיטימיות ולא תשובה אחת נכונה |
| **קשר לתוכן** (`content_connection`) | האם השאלה מחייבת לימוד של תכנים ספציפיים מהתוכנית |
| **אותנטיות** (`authenticity`) | האם מדובר בבעיה אמיתית מהעולם האמיתי — לא תרגיל מדומה |
| **מותאמת לגיל** (`age_appropriate`) | האם רמת המורכבות והמושגים מתאימים לשכבת הגיל |
| **מתח ודילמה** (`tension_dilemma`) | האם יש מחלוקת, דילמה אתית, או ניגוד אינטרסים אמיתי |
| **בינתחומית** (`interdisciplinary`) | האם השאלה מחייבת שילוב של יותר מתחום דעת אחד |
| **חקירה עצמאית** (`independent_inquiry`) | האם התלמידים יכולים לכוון את החקירה בעצמם |
| **תוצר משמעותי** (`meaningful_product`) | האם ניתן לייצר תוצר שמגיע לקהל אמיתי מחוץ לכיתה |
| **מידע זמין** (`information_available`) | האם תלמידים יכולים למצוא מידע רלוונטי — לא סוד מקצועי |
| **לא גוגלאבל** (`not_googleable`) | האם אי אפשר לענות על השאלה עם חיפוש מהיר או שאלה ל-AI |

ציון כולל (`overall_score`) הוא ממוצע מדויק של עשרת הקריטריונים.

## מצב הדגמה

הכלי עובד ללא מפתח API — במצב הדגמה. ללא `ANTHROPIC_API_KEY`, הכלי מחזיר נתוני דוגמה עם השהיה של 2 שניות (לדימוי קריאה אמיתית). ניתן לבדוק את כל הממשק, הזרימות, והתצוגות בלי עלות.

---

## התקנה מהירה

```bash
# שיבוט ותלויות
git clone <repo-url>
cd PBLQuestionLab
npm install

# הגדרת סביבה
cp .env.local.example .env.local   # ראה קובץ לדוגמה, או הגדר ידנית
# ערוך .env.local עם המשתנים הנדרשים (ראה טבלה למטה)

# הרצה מקומית
npm run dev
```

פתח את [http://localhost:3000](http://localhost:3000).

## משתני סביבה

| משתנה | תפקיד | חובה? | ברירת מחדל |
|-------|--------|--------|------------|
| `ANTHROPIC_API_KEY` | מפתח API של Claude (Anthropic) | לא (מצב הדגמה בלעדיו) | — |
| `CLAUDE_MODEL` | מזהה מודל Claude לשימוש | לא | `claude-sonnet-4-6` |
| `DATABASE_URL` | מחרוזת חיבור PostgreSQL לארכיון | לא (ארכיון לא יפעל) | — |
| `DELETE_PIN` | קוד PIN למחיקת שאלות מהארכיון | לא (מחיקה לא תעבוד) | — |

**הערה**: `ANTHROPIC_API_KEY` לא נשמר בקוד ולא מופיע בתגובות ה-API.

## פריסה

### Railway

הפרויקט כולל `railway.json` מוגדר מראש:

```bash
# ב-Railway Dashboard:
# 1. חבר את ה-repository
# 2. הוסף שירות PostgreSQL (אוטומטי)
# 3. הגדר משתני סביבה: ANTHROPIC_API_KEY, DELETE_PIN
# 4. Railway יבנה ויפרוס אוטומטית
```

### Render

הפרויקט כולל `render.yaml` מוגדר מראש עם PostgreSQL מקושר:

```bash
# ב-Render Dashboard:
# 1. חבר את ה-repository — Render יזהה את render.yaml
# 2. הגדר ידנית: ANTHROPIC_API_KEY (לא מסונכרן מטעמי אבטחה)
# 3. DELETE_PIN (אופציונלי)
# 4. DATABASE_URL מקושר אוטומטית מה-PostgreSQL service
```
